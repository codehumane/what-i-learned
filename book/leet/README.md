
# 무엇이 1등 팀을 만드는가?

> Leading Effective Engineering Teams

## CH 01. 무엇이 효과적인 팀을 만드는가?

### 1.2 동기부여가 성과를 이끌어낸다

- 내적 동기부여 중요성 이야기
- 여기에는 자율성, 전문성, 목적의식 필요 (다니엘 H. 핑크, 드라이브 인용)
- 데이비드라는 엔지니어를, 수익 보다는 사회적 영향력에 목표를 둔 프로젝트로 이동시켜, 식었던 열정을 되살리기 (목적의식 사례)
- 팀원에게 개별 모듈 개발을 주도하게 해서 자율성과 주인의식 높이도록 (자율성 사례)
- 팀원에게 최신 도구를 접할 수 있게 해 업무에 필요한 기술을 능숙하게 하도록 (전문성 사례)
- 팀원의 기여가 조직 목표에 어떻게 직접적으로 영향을 미치는지 명확히 설명 (목적의식 사례)

### 1.3 효과적인 팀 만들기

#### 1.3.1 적합한 인재 모으기

**적절하게 우선순위를 정하고 독립적으로 업무를 수행**

- 조직과 프로젝트의 전략적 우선순위를 명확하게 공유
- 이를 통해 팀원들이 업무 우선순위를 설정하는 데 도움을 줄 수 있음
- 정기적 체크인 미팅과 목표 설정 회의를 통해 꾸준히 도움을 주고,
- 엔지니어들이 각자 맡은 업무를 효과적으로 처리할 수 있도록 필요한 자원과 지침을 언제나 활용할 수 있도록 준비

**다양성과 포용성**

중요하고 당연한 얘기들 계손 나오는 가운데, 아래 사례가 최근의 고민 중인 내용이라 기록.

> 하지만 초기 회의는 대화보다는 독백에 가까웠습니다. 똑똑하지만 경험이 부족한 신참들은 의견을 말하길 주저했고, 겨우 낸 의견도 경험 많은 이들의 목소리에 묻혀 머리기도 했습니다. 또한, 그들에게 "선배 엔지니어들이 말씀하시는 것들에 비하자면 제가 말하는 내용들은 중요하지 않은 것 같습니다." 같은 말을 주기적으로 듣기도 했습니다. 저는 이런 문제를 해결하기 위한 방안을 마련해야 했습니다.

이를 해결하기 위해 아래와 같은 방안을 만들어 진행했다고 함.

1. 라운드 로빈 회의 도입 (모두가 동등하게 자신의 생각과 의견 말하는 시간 할당)
2. 격주로 가상의 '문화 교류' 모임 진행 (각자의 문화에만 존재하는 독특한 것들을 공유하며 상호 존중의 바탕 마련)
3. 열린포럼을 열어 심리적 안전감 증진 (익명으로 생각과 걱정을 포스팅하고, 매주 팀 미팅에서 사소한 의견도 경청하고 고민함 -> 내성적인 사람들도 점점 아이디어를 용기있게 냄)
4. 신참 팀원과 베테랑 멘토 짝지어, 조직의 역학 관계 알려주고 업계 규범을 가르치고 경력 개발을 위한 소프트 스킬도 키움

#### 1.3.2 팀 정신 일깨우기

- 집단이 공동의 목표를 달성하기 위해 노력하고 서로 도울 때 팀 정신을 가졌다고 표현
- 이를 위해 3가지가 필요한데, 그 중 하나는 팀 내 역할과 책임을 명확히 하기
  - 사례로, 프로젝트에서 개발자들에게 문서와 단위 테스트를 직접 작성해야 함을 명시해 문제 개선
- 또 하나 필요한 건 공동의 목적
  - 목표는 팀이 달성해야 하는 것
  - 목적은 목표를 왜 달성해야 하는지에 대한 것
  - 아키텍트는 확장성과 성능, 개발자는 마감 기한과 코드 효율성, 디자이너는 미학과 사용자 경험에 대한 목표를 가짐
  - 만약 공동의 목적이 없으면 갈등, 사용성 문제, 은근히 늘어나는 업무 범위, 기술 부채 등을 마주하게 됨
  - 목적을 위해서는 프로젝트 목적, 대상 사용자, 기대하는 영향력을 명확하게 설명해야 함
  - 또한 주기적으로 각자의 전문성과 우려사항을 나누며 정기적인 체크인이나 워크샵 진행
- 다음으로 필요한 건 팀원 간의 신뢰 구축
  - 지식을 나누고 비난 없이 서로 도움을 요청해서, 문제가 있더라도 신속하게 해 나가고 사기가 높음
  - 또 다른 한 팀은 베테랑 개발자들로 구성되었지만 서로 신경 쓰지 않고 맡은 모듈에만 집중하여 협업 어려움
  - 신뢰를 위해 열린 소통과 피드백 주고 받기 장려, 서로 개인적으로 알아갈 기회 제공, 프로젝트 목표와 일정 투명하게 공유, 팀워크와 협업 장려

#### 1.3.3 효과적으로 리더십 발휘하기

리더 책임으로 아래 3가지 이야기.

1. 팀에 영감을 불어넣기
2. 영향력 행사하기
3. 공동의 목표를 향하도록 안내하기

그리고 팀의 성과와 그 성과가 비즈니스에 미치는 영향에 대해 조직의 내외부 이해관계자들과 공유하는 활동으로 전략적 가시성<sup>strategic visibility</sup> 소개.

1. 더 큰 목표에 일치시키기 - 팀의 업무를 조직의 주요 목표와 일치시키기 (조직의 프로젝트 중 팀의 역량을 잘 발휘할 수 있는 것에 집중한 사례 언급)
2. 팀의 고유한 강점을 이용 - 팀원의 고유한 기술과 지식을 파악 후 적극적 활용
3. 효과적으로 소통하기 - 정기적으로 진행 상황과 배운 것들을 공유 (투명성은 업무에서의 인정과 원활한 협력을 가져옴)

## CH 02. 효율성 vs 효과성 vs 생산성

### 2.1 효율성, 효과성, 생산성의 차이점

#### 2.1.1 목표

| | 효율성 | 효과성 | 생산성 |
| -- | -- | -- | -- |
| 목표 | 일을 올바르게 수행하기 | 올바른 일을 하기 | 투입 대비 결과물 |
| 측정법 | 시간 | 고객 만족도 | 코드 줄 수 |
|      | 리소스 활용도 | 비즈니스 가치 | 기능 점수 부여 |
|      | 품질 | 사용자 채택률 | 스토리 포인트 부재 |

왜의 중요성에 관한 가상의 사례 이야기 소개.

- 회사의 주력 소프트웨어를 마이크로서비스와 쿠버네티스로 이전을 맡음
- 비즈니스에 중요한 작업을 하면서 새로운 기술을 배울 수 있는 기회로 설렘
- 분위기도 좋고 일정 문제 없이 스프린트 진행해 나감
- 하지만 사용자 피드백에서는 성능 개선도, 사용 편의성 향상도 없었음
- 데이터 기반의 리더는 지표 확인 후, 영향력 없는 기술의 왕국을 구축하고 있음을 알게 됨
- 더 빠르게 빌드하고 배포 수 있었지만, 사용자에게 도움 되는 기술적 고려사항은 활용하지 못함
- 효율성과 생산성에 대한 적절한 정의와 지표 개발에 집중해야 함
